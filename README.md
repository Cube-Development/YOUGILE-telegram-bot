# YouGile Telegram Bot

Многофункциональный Telegram-бот для управления задачами на досках YouGile прямо из мессенджера. Бот парсит команды, поддерживает реплаи (ответы на сообщения) для привязки действий к конкретным задачам и кеширует структуру доски для мгновенного отклика.

## Особенности и функционал

Бот предоставляет полный CRUD-интерфейс для работы с задачами:

* **Создание задач** (`/new` или `/task_new`) — позволяет создать задачу, указав целевую колонку, заголовок, описание и исполнителей (одного или нескольких через пробел).
* **Список задач** (`/list` или `/task_list`) — выводит задачи из указанной колонки. Поддерживает фильтрацию по пользователю (исполнителю).
* **Редактирование задач** (`/edit` или `/task_edit`) — вызывается как *реплай* на сообщение с задачей. Можно точечно изменить заголовок, описание, исполнителей (можно указать нескольких) или перенести в другую колонку.
* **Перемещение задач** (`/move` или `/task_move`) — быстрый перенос задачи в другую колонку (реплай).
* **Удаление задач** (`/delete` или `/task_delete`) — удаляет задачу (реплай).

### Дополнительные команды

* `/start` — Инициализирует работу с ботом.
* `/columns` — Загружает и показывает список колонок доски.
* `/instructions` — Выводит подробную справку по работе с командами.
* `/templates` — Отправляет готовые текстовые шаблоны для копирования.

## Технический стек

* **Язык**: TypeScript
* **Фреймворк**: Telegraf
* **Тестирование**: Vitest
* **HTTP клиент**: Axios
* **Кеширование**: `lru-cache`

## Архитектура проекта

Проект спроектирован по принципам модульности, разделения ответственности (Separation of Concerns) и DRY:

* `src/modules/telegram/commands` — реализация всех команд бота (создание, редактирование, просмотр).
* `src/shared/telegram/utils` — переиспользуемые утилиты:
  * Парсеры сообщений (`parsers.ts` / `parseCommandFields`).
  * Генерация HTML-шаблонов ответов (`templates.ts`).
  * Резолвинг пользователей и колонок (`resolve.ts`).
* `src/shared/cache` — модуль кеширования колонок доски.
* `src/constants` — константы проекта, включая маппинг Telegram-юзернеймов на YouGile ID (`users.map.ts`).
* `src/shared/api/generated` — автоматически генерируемые API-клиенты и DTO-модели.
* `src/scripts/generate-api.ts` — самописный генератор TypeScript-типов из OpenAPI спецификации `api-docs.json`.

## Установка и запуск

1. Установите зависимости:

   ```bash
   npm install
   ```

2. Создайте файл `.env` в корне проекта на основе необходимых переменных окружения:

   ```env
   BOT_TOKEN="твой_телеграм_токен"
   YOUGILE_BASE_URL="https://ru.yougile.com/api-v2"
   YOUGILE_KEY_COMPANY="твой_ключ_yougile"
   YOUGILE_BLOGIX_PROJECT_ID="id_проекта"
   YOUGILE_BLOGIX_BOARD_ID="id_доски"
   USERS_MAP="telegram_username:yougile_user_id"
   ```

3. Запуск бота в режиме разработки:

   ```bash
   npm run dev
   ```

4. Запуск тестов:

   ```bash
   npm run test
   ```

## Доступные NPM-скрипты

* `npm run dev` — запуск с автоперезагрузкой (nodemon + tsx).
* `npm run start` — запуск без автоперезагрузки (production-like).
* `npm run test` — запуск unit-тестов Vitest (покрыта вся логика парсеров и шаблонизации).
* `npm run codegen` — запуск скрипта кодогенерации API (`generate-api.ts`).
* `npm run lint` / `npm run format` — проверка и форматирование кода.

## Как добавить нового пользователя (права доступа)?

Бот использует `authMiddleware` для защиты: только авторизованные пользователи могут выполнять команды.
Кроме того, этот маппинг нужен, чтобы корректно распознавать исполнителей (`assigned: @user1 @user2`).

Для предоставления доступа и привязки логина добавьте запись в переменную `USERS_MAP` в файле `.env`.
Формат: `username:id,username2:id2`

```env
USERS_MAP="telegram_username:yougile_id-123456789,ivanov:other_id-0000"
```

*Где `telegram_username` — имя пользователя в Telegram (без `@`), а `yougile_id` — ID пользователя в системе YouGile.*
